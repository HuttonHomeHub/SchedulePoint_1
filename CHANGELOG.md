## [Unreleased]

### Added

- Repository engineering foundation: Turborepo + pnpm monorepo layout
  (`apps/web`, `apps/api`, `packages/config`, `packages/types`).
- Tooling: TypeScript (strict), ESLint (flat config), Prettier, EditorConfig,
  commitlint, Husky hooks, and lint-staged.
- Testing scaffolding: Vitest, Supertest, and Playwright configuration.
- Docker foundation: multi-stage Dockerfiles for `web` and `api`, plus a
  local `docker-compose.yml` with PostgreSQL.
- CI/CD: GitHub Actions for CI, CodeQL, releases (Changesets), and container
  publishing to the GitHub Container Registry.
- Project governance: issue/PR templates, CODEOWNERS, Dependabot, security
  policy, code of conduct, and contributing guide.
- Documentation set under `docs/` (architecture, API, design system, testing,
  deployment, development, roadmap, backlog, decisions, tech debt) and ADRs.
- Frontend architecture, design system (validated AA tokens), UX/component/
  quality standards, and ADRs 0004–0007.
- Backend architecture (modular monolith): API/database/security/observability/
  performance standards, ADRs 0008–0013, and a working **reference feature
  template** (`apps/api/src/modules/reference/`) demonstrating module structure,
  validation, REST/OpenAPI, database standards (soft delete, auditing,
  optimistic locking), RBAC with resource scoping, structured logging, and unit
  + API (Supertest) tests.
- Specialised Claude agents for frontend and backend design/review.
- Delivery process (`docs/PROCESS.md`): idea → business understanding →
  requirements → technical analysis → solution design → implementation plan →
  approval → build, with feature-spec/implementation-plan templates, a worked
  example, Definition of Ready/Done, and a `feature-analyst` agent. PR template
  updated with the Feature Completion Criteria.
- Project operating manual: `CLAUDE.md`.

### Changed

- **Repository readiness review** before application development: the reference
  feature is now a **non-shipping template** (`apps/api/examples/reference-feature/`,
  ADR-0014) — removed from the app module, build, and migrations; the live schema
  has no models (the first feature writes the first migration). The RBAC model
  was made feature-agnostic. Docs (ROADMAP, BACKLOG, TECH_DEBT, `.env.example`)
  were refreshed to remove completed/stale items. The initial version remains
  **0.1.0** (foundations only).
- **Reference template elevated to the canonical implementation standard**
  (ADR-0015): added the repository/data-access layer (controller → service →
  repository), expanded `docs/REFERENCE_FEATURE.md` into the full standard
  (backend + frontend, with when/how/customise/consistent/mistakes guidance),
  and made it **CI-verified** via `scripts/verify-template.sh` and a "Verify
  feature template" CI job (materialise → type-check → unit-test) so it can't
  rot. CLAUDE.md now mandates template-driven feature development.
- **Generalised into a domain-neutral base ("Blank App").** Renamed the
  workspace (`bills` → `blank-app`) and package scope (`@bills/*` → `@repo/*`),
  generalised resource scoping from "household" to "organisation", replaced
  product docs (README, ROADMAP, BACKLOG, worked example) with neutral
  equivalents, and reframed domain assumptions (e.g. money-as-minor-units) as
  conditional guidance. See `docs/DECISIONS.md`.
